# Платформа для поиска мастеров

Веб-платформа на Django для связи заказчиков с исполнителями различных услуг (ремонт, обучение, красота, IT-услуги и др.).

## Описание проекта

Платформа предоставляет возможность:
- **Заказчикам**: создавать заказы, искать исполнителей, просматривать портфолио, оставлять отзывы
- **Исполнителям**: откликаться на заказы, управлять портфолио, получать рейтинги
- **Администраторам**: модерировать контент, управлять пользователями, просматривать статистику

## Основные функции

### Для заказчиков
- Регистрация и управление профилем
- Создание и редактирование заказов
- Поиск и фильтрация заказов по категориям, бюджету, дате
- Просмотр откликов от исполнителей
- Оценка работы исполнителей (рейтинг 1-5 звезд)
- Система уведомлений

### Для исполнителей
- Регистрация с указанием специализации
- Управление портфолио (добавление работ с фото)
- Поиск подходящих заказов
- Отклики на заказы с предложением цены
- Просмотр своего рейтинга и отзывов
- Ответы на отзывы клиентов

### Административная панель
- Управление пользователями (блокировка)
- Модерация контента
- Просмотр статистики платформы
- Управление категориями услуг

## Установка и запуск

### Требования
- Python 3.8 или выше
- pip
- virtualenv (рекомендуется)

### Шаги установки

1. Клонируйте репозиторий:
```bash
git clone https://github.com/AlexandrVdovenko/masters-platform.git
cd masters-platform
```

2. Создайте виртуальное окружение:
```bash
python -m venv venv
```

3. Активируйте виртуальное окружение:
- Windows:
```bash
venv\Scripts\activate
```
- Linux/Mac:
```bash
source venv/bin/activate
```

4. Установите зависимости:
```bash
pip install -r requirements.txt
```

5. Примените миграции:
```bash
python manage.py migrate
```

6. Загрузите начальные данные (категории):
```bash
python manage.py loaddata categories
```

7. Создайте суперпользователя:
```bash
python manage.py createsuperuser
```

8. Запустите сервер разработки:
```bash
python manage.py runserver
```

9. Откройте браузер и перейдите по адресу: http://127.0.0.1:8000/

## Безопасность

Проект использует следующие меры безопасности:
- Хеширование паролей (PBKDF2)
- CSRF защита
- XSS защита через auto-escaping
- SQL injection защита через Django ORM
- Валидация и санитизация пользовательского ввода
- Ограничение размера загружаемых файлов
- Таймаут сессии (60 минут)

## API Endpoints

### Аутентификация
- `POST /accounts/register/` - Регистрация
- `POST /accounts/login/` - Вход
- `POST /accounts/logout/` - Выход
- `GET/PUT /accounts/profile/` - Профиль пользователя

### Заказы
- `GET /orders/` - Список заказов (с фильтрами)
- `POST /orders/` - Создание заказа
- `GET /orders/<id>/` - Детали заказа
- `PUT /orders/<id>/` - Обновление заказа
- `DELETE /orders/<id>/` - Удаление заказа

### Отклики
- `POST /orders/<id>/responses/` - Создание отклика
- `GET /orders/<id>/responses/` - Список откликов
- `POST /responses/<id>/accept/` - Принять отклик
- `POST /responses/<id>/reject/` - Отклонить отклик

### Портфолио
- `GET /portfolio/<user_id>/` - Портфолио исполнителя
- `POST /portfolio/` - Добавление работы
- `PUT /portfolio/<id>/` - Обновление работы
- `DELETE /portfolio/<id>/` - Удаление работы

### Отзывы
- `GET /reviews/<executor_id>/` - Отзывы исполнителя
- `POST /reviews/` - Создание отзыва
- `PUT /reviews/<id>/` - Обновление отзыва

### Уведомления
- `GET /notifications/` - Список уведомлений
- `PUT /notifications/<id>/read/` - Отметить прочитанным

## Развертывание

### Production настройки:
1. Установите `DEBUG = False` в settings.py
2. Настройте `ALLOWED_HOSTS`
3. Используйте PostgreSQL вместо SQLite
4. Настройте HTTPS
5. Используйте Gunicorn как WSGI сервер
6. Настройте Nginx как reverse proxy
7. Настройте переменные окружения для SECRET_KEY

### Пример с Gunicorn:
```bash
gunicorn masters_platform.wsgi:application --bind 0.0.0.0:8000
```
