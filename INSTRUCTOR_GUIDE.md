# Руководство для преподавателя

## Обзор курсовой работы

Данная курсовая работа представляет собой веб-платформу для поиска мастеров, разработанную с использованием современных методологий программной инженерии.

## Ключевые особенности проекта

### 1. Методология Spec-Driven Development
Проект выполнен с использованием формальной спецификации:
- **requirements.md** - 12 требований в формате EARS
- **design.md** - архитектура + 44 correctness properties
- **tasks.md** - пошаговый план реализации (80+ задач)

### 2. Property-Based Testing
Применен современный подход к тестированию с использованием библиотеки Hypothesis:
- 44 универсальных свойства системы
- Автоматическая генерация тестовых данных
- Проверка инвариантов и round-trip свойств

### 3. Полная документация
- Техническая документация
- Руководства пользователя
- Инструкции по установке и развертыванию

## Быстрая проверка работоспособности

### 1. Установка (5 минут)
```bash
git clone https://github.com/AlexandrVdovenko/masters-platform.git
cd masters-platform
python -m venv venv
venv\Scripts\activate  # Windows
pip install -r requirements.txt
python manage.py migrate
python manage.py loaddata fixtures/categories.json
python manage.py createsuperuser
python manage.py runserver
```

### 2. Проверка функциональности
- Откройте http://127.0.0.1:8000/
- Зарегистрируйтесь как заказчик и исполнитель
- Создайте заказ
- Сделайте отклик
- Проверьте админ-панель: http://127.0.0.1:8000/admin/

### 3. Проверка кода
```bash
# Запуск тестов (если реализованы)
pytest

# Проверка структуры
python manage.py check
```
